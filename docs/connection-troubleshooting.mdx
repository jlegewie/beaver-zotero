# Connection troubleshooting

If Beaver shows **"Connection Failed — Could not connect to the server"**, it means Beaver couldn't reach its backend. In most cases the problem is between Zotero and the internet, not with Beaver's servers. This page walks through the common causes and what to try, and gives your IT team the details they need if you're on a work or school network.

## What Beaver needs to connect

Beaver connects to two services from inside Zotero:

- **The Beaver API** at `api.beaverapp.ai` on port **443** — over both **HTTPS** *and* **WebSocket Secure (`wss://`)**. Beaver streams responses over WebSocket, so allowing HTTPS alone is not enough.
- **Supabase** at `*.supabase.co` on port **443**, over HTTPS. This is used for sign‑in.

If either of these is blocked, or if the connection is intercepted and re‑signed by security software, you'll see "Connection Failed".

## Most common cause: your network is blocking Beaver

Work, school, and hotel Wi‑Fi networks often run a **proxy**, **firewall**, or **content filter** that blocks the kind of streaming connection Beaver uses. This is by far the most common cause of "Connection Failed" reports.

**Quick test:** try Beaver on a different network — a personal hotspot from your phone is the fastest way. If it works there but fails on your work network, your network is the culprit.

**What to do:**

1. If you can switch networks, do so.
2. If you have to stay on the restricted network, ask your IT team to allow Beaver's traffic — see [For IT administrators](#for-it-administrators) below.
3. If your organisation requires a specific HTTP proxy, Zotero (which is based on Firefox) honours proxy settings under **Edit → Preferences → Advanced → Config Editor**, in the `network.proxy.*` keys.

## Security software interference

Antivirus, VPN, or corporate security suites sometimes intercept every secure connection your computer makes, decrypt it, and re‑sign it with their own certificate. That process — often called **TLS inspection** or **SSL inspection** — breaks Beaver's connection, because Zotero doesn't trust the re‑signed certificate.

If Beaver's error shows **error code 1015**, TLS inspection is the likely cause.

**What to do:**

1. Temporarily disable your VPN or the "web protection" feature of your antivirus, and try again. If Beaver works, that program is the cause.
2. Add an exception for `api.beaverapp.ai` in your security software so its traffic isn't inspected.
3. On a corporate machine, your IT team is the right point of contact — see below.

## Temporary server issues

If Beaver's error shows a code in the **1011–1014** range or a **4xxx** code, the server is telling Beaver something went wrong on its side. This is almost always temporary — wait a moment and click **Retry**.

If it keeps failing for more than a few minutes and switching networks doesn't help, please [file a report](#if-nothing-works).

## For IT administrators

If a colleague at your organisation is having trouble using Beaver, please allow the following outbound traffic on TCP **port 443**:

| Host | Protocols |
|------|-----------|
| `api.beaverapp.ai` | HTTPS **and** WebSocket Secure (`wss://`) |
| `*.supabase.co` | HTTPS |

**Requirements:**

- **WebSocket traffic must be permitted.** Beaver streams responses over `wss://`, not plain HTTPS. Proxies that strip or block the `Upgrade: websocket` and `Connection: Upgrade` request headers will cause "Connection Failed".
- **Exempt `api.beaverapp.ai` from TLS/SSL inspection.** Many web filters and next‑gen firewalls (Zscaler, Netskope, Blue Coat, Palo Alto, Fortinet, etc.) inspect encrypted traffic by re‑signing it with an internal certificate. Zotero will not trust that certificate and the connection will fail.
- **No custom API endpoint or proxy setting is required inside Beaver.** Beaver uses whatever proxy is configured for Zotero (Edit → Preferences → Advanced → Config Editor, `network.proxy.*`).

If you need more information, please contact us at [support@beaverapp.ai](mailto:support@beaverapp.ai).

## Reading the error code

Beaver shows a numeric error code below the "Connection Failed" message. If you're filing a support ticket, including this code helps us diagnose the problem quickly. You may also see one of two suffixes:

- **`https ok`** — Beaver reached its servers over plain HTTPS, so only the real‑time (WebSocket) traffic is blocked. This almost always points at a proxy or filter on your network that allows regular web traffic but not WebSocket.
- **`https blocked`** — Neither WebSocket nor regular HTTPS worked, so Beaver is unreachable from your network as a whole. Check your internet connection, then any VPN or security software that might be blocking `api.beaverapp.ai`.

| Code | Meaning | Common cause |
|------|---------|--------------|
| **1005** | No status received | Proxy or firewall dropping the connection |
| **1006** | Connection blocked before Beaver's server responded | Corporate/school proxy, firewall, or filter blocking WebSocket traffic |
| **1011** | Server error | Temporary problem on Beaver's servers — retry in a moment |
| **1012** | Service restart | Beaver's servers are restarting — retry in a moment |
| **1013** | Try again later | Beaver's servers are temporarily overloaded — retry in a moment |
| **1014** | Bad gateway | Temporary problem upstream of Beaver's servers — retry in a moment |
| **1015** | TLS handshake failure | TLS/SSL inspection by antivirus, VPN, or corporate proxy |
| **1002 / 1003 / 1007 / 1008 / 1009 / 1010** | Protocol error | Usually resolved by updating Beaver to the latest version |
| **4000–4999** | Server‑defined rejection | Almost always temporary — retry, or check that Beaver is up to date |

## If nothing works

If you've tried a different network, checked your security software, and still see "Connection Failed", please send us a debug log so we can help:

1. In Zotero, go to **Help → Debug Output Logging → Enable**.
2. Reopen Beaver and reproduce the error.
3. Go to **Help → Debug Output Logging → View Output**.
4. Copy the log — especially any lines starting with `[Beaver]` — into an email to [support@beaverapp.ai](mailto:support@beaverapp.ai), together with the error code shown in the Beaver panel.

We'll follow up with next steps.
